# v1.4 变更记录

**版本**：v1.4  
**状态**：已封板（2026-03-26）  
**版本名称**：AI 专题试运行版

## 1. 版本目标
在不影响 `v1.2` 下载归档主链路的前提下，将 AI 专题能力从冻结样本验证升级为真实归档小范围试运行可用。

## 2. 本版新增（已完成）
- `src/analysis_v14/` 独立离线分析模块与运行入口：`python -m src.analysis_v14.run`。
- 新增 `mock/real` 双模执行，`real` 模式支持 OpenAI 兼容接口。
- 解析层升级：`docx/pdf/doc` 统一解析链路，失败进入 `review_queue`。
- 标准化失败原因码：`PARSE_FAILED_DOC`、`PARSE_FAILED_PDF`、`PARSER_TOOL_MISSING`、`MODEL_CALL_FAILED`。
- 标准产物保持兼容并增强字段：`run_id`、`model_mode`、`model_name`、`parse_status`。
- 新增 `data/input/v1.4/` 输入基线目录与复核模板。
- 新增 `tests/test_analysis_v14.py` 覆盖 v1.4 关键路径与失败兜底。
- 新增业务可读输出：`AI专题业务验收简报`、`AI专题看板.html`、周/月与趋势看板CSV、机会池与溯源工作表。

## 3. 本版未做（明确边界）
- 不做在线服务化网页系统（仅生成离线可查看 HTML 报告）。
- 不做数据库正式实现。
- 不做多专题统一平台。
- 不做经营结果数据关联。
- 不改 `v1.2` 主链路职责（`src/main.py`）。

## 4. 已知限制
- `doc` 解析依赖系统工具链（`antiword/catdoc`），不同环境可用性可能不同。
- `real` 模式效果受模型与提示词质量影响，仍需通过首轮试运行校正。
- 当前仍为离线批处理，不提供在线交互式复核页面。

## 5. 后续迭代方向
- 基于试运行误判/漏判结果优化解析与标签判定。
- 在保证追溯与准确前提下逐步降低人工复核成本。
- 在 `v1.5` 完成 real 模式质量门槛与长期趋势口径固化后再推进更强产品化展示层。
