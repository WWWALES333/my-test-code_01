# AI 专题试运行版技术方案

**版本**：v1.4  
**状态**：正式基线  
**阶段**：开发前确认版

## 1. 版本说明
`v1.4` 技术方案只承接 `PRD.md` 已锁定范围：  
在不影响 `v1.2` 下载归档主链路的前提下，把 `v1.3` 离线分析升级为真实归档小范围试运行可用。

## 2. 当前项目现状
- `src/main.py` 已稳定承担邮件拉取、下载、归档、审计、调度和通知。
- `src/analysis_v13/` 已具备离线专题分析基础链路与标准产物输出。
- 当前 `v1.3` 对 `pdf/doc` 解析仍是“未启用解析 -> 进入 review_queue”。

## 3. 为什么不推翻重写
- 下载归档链路已在真实业务中验证，重写会引入高回归风险。
- `v1.4` 核心问题是分析可用性与试运行质量，不是接入链路重构。
- 最优策略是复用既有底座，增量升级解析、判定和复核闭环。

## 4. 首期系统形态（v1.4）
- 运行形态：离线批处理，不做在线服务。
- 判定形态：`mock/real` 双模并行，`real` 使用 OpenAI 兼容接口。
- 数据形态：结构化文件产物 + 人工复核文件产物。
- 质量形态：准确率优先，不确定允许进入正式复核链路。

## 5. 离线执行入口设计
入口保持单一，最小参数集固定：
- `--samples`
- `--annotations`
- `--out`
- `--model-mode`

执行能力要求：
- 支持 `v1.3` 冻结样本回归。
- 支持 `2026` 真实归档试运行。
- 同一输入多次执行产物口径一致，可复查可比对。

## 6. 处理流水线
建议固定流水线顺序：
1. 输入扫描与文档元数据整理  
2. 正文抽取（含 `docx/pdf/doc`）  
3. 片段切分与片段清洗  
4. 规则初筛  
5. `mock/real` 精判  
6. 结构化产物写出  
7. `review_queue` 生成与复核表输出  
8. 专题摘要生成

## 7. 数据对象与输出契约
本期沿用 `v1.3` 核心契约并保持兼容：
- `report_index.jsonl`
- `tag_result.jsonl`
- `evidence_span.jsonl`
- `review_queue.jsonl`
- `review_queue.csv`
- `AI专题摘要.md`

状态口径固定：
- `confirmed`
- `uncertain`
- `pending_human_review`

## 8. 输入输出目录设计
输入目录：
- `data/input/v1.3/`（冻结基线）
- `data/input/v1.4/`（试运行样本、批次、复核记录）

输出目录：
- `data/output/insights/v1.4/extracted/`
- `data/output/insights/v1.4/reports/`
- `data/output/insights/v1.4/review/`（如复核记录独立）

约束：
- 不覆盖 `v1.3` 产物。
- 不混用历史实验目录。

## 9. `review_queue` 设计
`review_queue` 是正式人工复核入口，非异常垃圾桶。  
进入队列的记录必须具备：
- 明确进入原因（解析失败、边界语义、口径冲突等）
- 可追溯证据与原文路径
- 可记录复核结论的字段基础

## 10. 代码模块落点
分析实现继续采用独立模块路径，不堆入 `src/main.py`：
- `src/main.py`：仅负责 `v1.2` 下载归档审计主链路
- `src/analysis_v14/`：承接 `v1.4` 离线分析能力

`v1.4` 模块建议最小分层：
- loader / parser / tagger / review / reporter / run

## 11. 模型接入设计（real 模式）
- `real` 模式以 OpenAI 兼容接口为统一适配层。
- 模型参数与密钥只走环境变量，不写入仓库配置文件。
- 接口失败需显式记录并触发降级策略（进入复核，不静默丢弃）。

## 12. 风险与折中
风险 1：`pdf/doc` 解析稳定性不足。  
折中：解析失败保留原因码并入复核，不阻塞整批运行。

风险 2：真实模型输出波动。  
折中：保留 `mock` 对照回归，关键口径由人工复核兜底。

风险 3：复核负担上升。  
折中：按原因码和优先级排序，先处理高价值片段。

风险 4：实现层面越界到平台化。  
折中：严格按 PRD 范围推进，本期不做网页/数据库/多专题平台。

## 13. 本期刻意不做
- 不改造 `v1.2` 下载归档主链路。
- 不建设正式 Web 页面。
- 不引入数据库正式实现。
- 不扩展多专题统一平台。
- 不接经营结果数据关联。
