# 销售周报自动下载与分类系统 PRD

**版本**：v1.2  
**日期**：2026-03-23  
**状态**：已封板归档

## 1. 版本目标
`v1.2` 的目标是把销售周报和月报的下载、归档、审计和调度稳定性补齐，形成可持续运行的数据底座。

## 2. 核心能力
- 连接阿里邮箱 IMAP 并检索周报、月报邮件
- 自动下载 `.doc`、`.docx`、`.pdf` 附件
- 从附件文件名解析时间并归档
- 基于内容哈希去重
- 输出运行日志和审计结果
- 通过定时任务和飞书通知实现自动运行

## 3. 关键业务规则
- 周报优先使用文件名中的 `X年X月第N周` 进行归档
- 月报优先使用文件名中的年份和月份进行归档
- 邮件发送日期只作为周报无显式周次时的兜底信息
- 周报完整性软阈值为每周 22 到 24 份
- 月报完整性阈值为每月 9 份

## 4. 验收结果
- 周报错分周问题已修复
- 下载链路支持重试和错误落盘
- 调度失败不再导致守护进程静默失效
- 历史数据基线收敛到 `2023` 年及以后

## 5. 后续衔接
`v1.2` 只负责数据底座，不负责 AI 专题分析。后续版本文档见 `docs/releases/v1.3/`。
