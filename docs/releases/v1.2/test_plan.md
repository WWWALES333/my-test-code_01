# 测试方案文档

**版本**：v1.2  
**日期**：2026-03-23  
**状态**：已封板归档

## 1. 测试目标
验证下载和归档链路在真实邮件场景下稳定可用，并确认周报、月报不会因解析或调度问题出现系统性漏抓。

## 2. 覆盖范围
- 周报和月报邮件识别
- 附件下载与去重
- 周报和月报时间解析
- 周报错分周审计
- 定时任务运行
- 飞书通知和失败重试

## 3. 关键场景
- 周报文件名明确写 `X月第N周` 时按文件名归档
- 月报文件名明确写 `X月月报` 时按月份归档
- 文件名无法解析时进入失败或审计记录，而不是误归档
- 下载历史已存在时应跳过重复附件
- 调度阶段异常时应写入运行日志

## 4. 验收阈值
- 周报数量每周不低于 22 份时视为正常
- 月报数量每月不低于 9 份
- 分类路径准确率为 100%
- 失败场景必须有中文日志或审计记录

## 5. 结果说明
`v1.2` 重点验证的是数据底座稳定性，不包含专题分析标签和展示验证。
