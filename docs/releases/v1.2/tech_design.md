# 技术方案文档

**版本**：v1.2  
**日期**：2026-03-23  
**状态**：已封板归档

## 1. 当前实现形态
当前系统以 `src/main.py` 为主入口，围绕一个下载器主流程完成：
- 邮件连接与检索
- 附件抓取
- 时间解析
- 文件归档
- 审计摘要
- 定时执行
- 飞书通知

## 2. 关键技术点
- 使用 IMAP 连接企业邮箱
- 使用 JSON 存储下载历史和运行日志
- 使用附件内容哈希做去重
- 使用邮件最新优先遍历和断线重连，降低最新周报漏抓风险
- 使用审计输出识别错分周、冲突和完整性缺口

## 3. 数据文件
- `data/input/config.json`：运行配置
- `data/output/runtime/downloaded_history.json`：下载去重历史
- `data/output/runtime/run_log.json`：运行日志
- `data/output/audit/reports/`：审计报告

## 4. 设计取舍
- 保留当前单文件主流程，优先保证运行稳定
- 不在 `v1.2` 引入数据库
- 不在 `v1.2` 开发专题分析页面
- 不推翻已验证的邮件接入和归档逻辑

## 5. 与后续版本的关系
`v1.2` 为 `v1.3` 提供数据底座。后续如果进入专题分析开发，应在此基础上渐进拆分解析、标签和展示层，而不是重写下载链路。
