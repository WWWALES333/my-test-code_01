# 销售周报自动下载与分类系统 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
自动化工具，从阿里邮箱自动下载销售部门的周报/月报邮件附件，并根据时间自动分类存档。

### 1.2 目标用户
中医互联网平台的市场/运营人员

### 1.3 核心痛点
- 每周/每月收到大量销售周报/月报邮件，没时间逐封阅读
- 手动下载附件麻烦，容易遗漏
- 周报/月报散落在邮箱各处，难以统一管理和分析

## 3. 业务规则

### 3.1 邮件识别规则
- 邮件主题包含"周报"或"月报"关键词
- 邮件带有 .docx 或 .doc 格式附件

### 3.2 区域/战区识别规则（已实现但未用于分类）
- 从附件文件名中匹配已知的区域/战区关键词
- 支持多级区域识别（如"将军汤 广西区域"）

**当前支持区域/战区关键词：**
- 一战区~七战区、线上战区、云管家、将军汤、天雄汤
- 广西区域、粤琼区域、华中区域、华东区域、华北区域、西南区域
- 冀蒙、江苏、浙江、黑龙江、湖北、四川、云南、河南、陕西、辽宁、吉林、福建、上海、粤海、甘宁、晋甘宁

### 3.3 周报时间解析规则
- 支持格式：`YYYY年M月第N周` 或 `YYYY年 M月第 N周`
- 简单日期格式：`YYYY.M.D`、`YYYY-MM-DD`
- 无法解析时使用邮件发件时间

### 3.4 月报时间解析规则
- 支持格式：`2026年1月`、`2026年 1月`、`2026.1月`
- 简写年份：`26年1月`、`26年一月`

### 3.5 分类存档目录结构（当前实际结构）
```
06 销售周报/
├── {年份}/
│   ├── {月份}月第{N}周/    # 周报目录
│   │   └── *.docx
│   └── {月份}月报/         # 月报目录
│       └── *.docx
```

### 3.6 去重规则
- 使用 MD5 哈希对附件内容进行指纹识别
- 已下载的邮件记录存储在 `downloaded_history.json`
