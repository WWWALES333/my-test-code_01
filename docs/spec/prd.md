**版本**: v1.2
**日期**: 2026-03-19

---

## 1. 产品概述

### 1.1 产品定位
自动化工具，从阿里邮箱自动下载销售部门的周报/月报邮件附件，并根据时间自动分类存档。

### 1.2 目标用户
中医互联网平台的市场/运营人员

### 1.3 核心痛点
- 每周/每月收到大量销售周报/月报邮件，没时间逐封阅读
- 手动下载附件麻烦，容易遗漏
- 周报/月报散落在邮箱各处，难以统一管理和分析

## 4. 业务规则

### 4.1 发送规律

| 类型 | 发送时间 | 发送人 | 预期数量 |
|------|----------|--------|----------|
| 周报 | 每周六/日/一 | 各战区/区域主管 | 22份/周 |
| 月报 | 每月28-31日 | 9位负责人 | 9份/月 |

**月报发送人（9位）**:
1. 一战区主管（白及）
2. 二战区主管（全蝎）
3. 三战区主管（常山）
4. 四战区主管（花椒）
5. 五战区主管（柴胡）
6. 六战区主管（附子）
7. 七战区主管（雷丸）
8. 线上战区主管（阿魏）
9. 云管家主管（金橘）

### 4.2 文件名格式

#### 周报格式
```
标准: "2026年3月第2周 将军汤 河南工作周报.pdf"
结构: {年份}年{月份}月第{周}周 {区域}工作周报
```

#### 月报格式
```
标准: "2026年七战区1月月报.pdf"
结构: {年份}年{战区}{月份}月月报
```

### 4.3 邮件识别规则

```
规则1: 周报识别
├── 条件A: 主题包含"周报" (不含"月报")
├── 条件B: 附件为.doc/.docx/.pdf
└── 命中: 判定为周报

规则2: 月报识别
├── 条件A: 主题包含"月报"
├── 条件B: 附件为.doc/.docx/.pdf
└── 命中: 判定为月报
```

### 4.4 时间解析规则

**核心原则**: 必须从文件名解析时间，不使用邮件发送日期作为fallback

#### 周报时间解析
```
正则: (\d{2,4})年(\d{1,2})月第(\d{1,2})周
示例: "2026年3月第2周" → 年=2026, 月=3, 周=2
```

#### 月报时间解析
```
正则: (\d{2,4})年?(\d{1,2})月月报
示例: "2026年1月月报" → 年=2026, 月=1
示例: "七战区1月月报" → 年=当前年, 月=1
```

## 5. 验收标准

### 5.1 核心指标

| 指标 | 最低要求 | 理想目标 |
|------|----------|----------|
| 周报下载数量 | ≥20份/周 | 22份/周 |
| 月报下载数量 | ≥8份/月 | 9份/月 |
| 区域覆盖率 | ≥90% | 100% |
| 时间解析准确率 | ≥95% | 100% |
| 分类正确率 | 100% | 100% |

### 5.2 功能验证

- [ ] 定时任务每周一正常运行
- [ ] 定时任务每月1号正常运行
- [ ] 飞书通知正确推送结果
- [ ] 去重功能正常工作

### 5.3 日志要求

每次运行记录:
- 运行时间
- 扫描邮件数
- 下载数（成功/跳过/失败）
- 新增文件列表
