## 3. 核心算法设计

### 3.1 时间解析算法

#### 周报时间解析

```
输入: filename (如 "2026年3月第2周 将军汤 河南工作周报.pdf")

步骤:
1. 使用正则提取: (\d{2,4})年(\d{1,2})月第(\d{1,2})周
2. 如果2位年份，转换为4位 (26 -> 2026)
3. 返回 {year, month, week}

示例:
- "2026年3月第2周" → {year: 2026, month: 3, week: 2}
- "26年3月第1周"   → {year: 2026, month: 3, week: 1}
```

#### 月报时间解析

```
输入: filename (如 "2026年七战区1月月报.pdf")

步骤:
1. 使用正则提取: (\d{2,4})年?(\d{1,2})月月报
2. 如果无年份，使用当前年
3. 返回 {year, month}

示例:
- "2026年1月月报" → {year: 2026, month: 1}
- "七战区1月月报" → {year: 2026, month: 1}
```

### 3.2 区域识别算法

```
输入: filename

步骤:
1. 加载区域关键词列表
2. 遍历关键词，查找文件名中的匹配
3. 返回第一个匹配的关键词

关键词优先级:
1. 战区 (一战区~七战区, 线上战区, 云管家)
2. 大区 (广西区域, 粤琼区域...)
3. 省份 (上海, 广东, 广西...)
```

### 3.3 去重算法

```
输入: file_content (bytes)

步骤:
1. 计算MD5哈希: md5(file_content)
2. 读取downloaded_history.json
3. 遍历已下载文件，检查MD5是否存在
4. 返回: is_duplicate (bool)
```

## 4. 核心模块设计

### 4.1 邮件搜索模块

**类**: `WeeklyReportDownloader`
**方法**: `search_weekly_report_emails()`

```
功能: 搜索符合条件的周报/月报邮件

实现:
1. 连接IMAP邮箱
2. 搜索所有邮件（倒序遍历）
3. 对每封邮件:
   - 检查主题是否包含"周报"或"月报"
   - 检查是否有附件
   - 提取附件信息
4. 返回邮件列表
```

### 4.2 时间解析模块

**类**: `WeeklyReportDownloader`
**方法**: `_extract_time_info(filename, report_type)`

```
功能: 从文件名解析时间信息

参数:
- filename: 附件文件名
- report_type: "weekly" 或 "monthly"

返回: {year, month, week} 或 {year, month}
```

### 4.3 分类存档模块

**类**: `WeeklyReportDownloader`
**方法**: `_get_output_path(filename, report_type, year, month, week)`

```
功能: 计算输出文件路径

参数:
- filename: 文件名
- report_type: 报告类型
- year: 年份
- month: 月份
- week: 周次 (周报) 或 None (月报)

返回: 完整文件路径

目录结构:
周报: {output_root}/{year}/{month}月第{week}周/{filename}
月报: {output_root}/{year}/{month}月报/{filename}
```

## 6. 错误处理

### 6.1 时间解析失败

```
情况: 文件名无法解析出时间

处理:
1. 记录日志: "时间解析失败，跳过文件: {filename}"
2. 跳过该文件，不下载
3. 继续处理下一封邮件
```

### 6.2 附件下载失败

```
情况: 附件下载失败

处理:
1. 记录错误日志
2. 标记失败数+1
3. 继续处理下一封邮件
```
